## Реализация библиотеки

### Конфигурации

Эталонная ограниченная FIFO-очередь «писатели → читатели» с естественным backpressure: переполнение тормозит быстрых продюсеров вместо роста памяти или потери данных.

- **Решаемые проблемы:** разграничение скоростей продюсеров и консюмеров при жёстком лимите памяти и гарантии отсутствия потерь.
- **Роль в исследовании:** базовая (reference) стратегия для сравнения с кольцевыми (drop-oldest / block-writer) и lock-free по пропускной способности, латентности, памяти и fairness.

Кольцевая очередь на статическом буфере с вытеснением самого старого элемента при заполнении.  
Продюсер никогда не блокируется; консьюмер может ожидать появления элементов.

##### Поведение
###### Опустошение и переполнение

| Аспект                                                 | Решение                                                                                                                                    |
| ------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------ |
| Поведение при заполнении (продюсер)                    | Новый элемент всегда принимается. Если очередь заполнена, самый старый элемент удаляется, освобождая место. Поток-продюсер не блокируется. |
| Поведение при пустой очереди (блокирующий консьюмер)   | Метод `take()` блокирует вызывающий поток до появления элемента. Поток засыпает на условной переменной `notEmpty`.                         |
| Поведение при пустой очереди (неблокирующий консьюмер) | Метод `tryTake()` немедленно возвращает пустое значение (`std::nullopt`), если очередь пуста, не блокируясь.                               |
| Очерёдность пробуждения консьюмеров                    | При вставке элемента, если очередь была пуста, пробуждается **один** ожидающий консьюмер (FIFO).                                           |

###### Потокобезопасность
| Аспект                       | Решение                                                                                                                                                  |
| ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Модель конкурентного доступа | **MPMC** – любое количество продюсеров и консьюмеров могут одновременно вызывать любые методы очереди.                                                   |
| Механизм синхронизации       | **Один общий мьютекс** + **одна условная переменная** `notEmpty` для ожидания элементов.                                                                 |
| Атомарность операций         | Каждая операция захватывает мьютекс, выполняет действия с индексами/счётчиком и освобождает мьютекс. Вставка при необходимости вытесняет старый элемент. |
| Справедливость ожидания      | **FIFO** для очереди ожидающих консьюмеров.                                                                                                              |

###### Прерывания
| Аспект                   | Решение                                                                                                                                      |
| ------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------- |
| Подверженные методы      | Только блокирующие методы консьюмера: `take()` и `take(timeout)`.                                                                            |
| Механизм отмены ожидания | Метод `take()` и `take(timeout)` принимают токен отмены (`std::stop_token`) или ссылку на атомарный флаг отмены.                             |
| Реакция на сигнал отмены | При поступлении сигнала отмены поток пробуждается, **не извлекает элемент** (очередь не меняется) и выбрасывает исключение `QueueCancelled`. |
| Сообщение об отмене      | Исключение `QueueCancelled` (от `std::runtime_error`). Второй вариант – возврат `std::nullopt`.                                              |

###### Асимптотики

| Операция                    | Время | Память |
| --------------------------- | ----- | ------ |
| `put(T)`                    | O(1)  | O(1)   |
| `take()`                    | O(1)  | O(1)   |
| `tryTake()`                 | O(1)  | O(1)   |
| `peek()`                    | O(1)  | O(1)   |
| `size()`                    | O(1)  | O(1)   |
| `capacity()`                | O(1)  | O(1)   |
| `empty()`                   | O(1)  | O(1)   |
| `full()`                    | O(1)  | O(1)   |
| **Общая память очереди**    | —     | O(N)   |

Динамическое хранение в `std::deque` (по умолчанию) или `std::list`; буфер **не преаллоцируется** — память выделяется по мере роста. Отличие от кольцевых конфигураций и предмет отдельного сравнения «аллокации vs преаллоцированное кольцо». FIFO: запись в хвост (`push_back`), чтение с головы (`pop_front`).

- При переполнении (`size == capacity`) блокируется поток продюсера до освобождения места; при пустой очереди блокируется поток консьюмера до появления сообщения.
- Одно чтение разблокирует одного продюсера; одна запись — одного консьюмера.
- Замещения и потерь нет → целостность данных гарантируется.
- `Send` из единственного потока-консьюмера при полном буфере → *deadlock* (вырожденный сценарий: единственный потребитель сам себя блокирует).
- Сложность: запись / чтение — амортизированно **O(1)**; память — **O(`capacity`)** плюс накладные контейнера.
- `std::deque` / `std::list` **не потокобезопасны даже в SPSC**: одновременные `push_back` и `pop_front` гонятся за общим состоянием (`size`, узлы, карта блоков), поэтому взаимное исключение требуется во всех сценариях конкуренции.

**race condition риски** (сценарии задаются числом потоков-продюсеров/консюмеров в бенчмарке)
| Сценарий | риск  |
| -------- | ----- |
| SPSC     | `Send` ↔ `Read` (общий контейнер не потокобезопасен) |
| SPMC     | `Send` ↔ `Read` + `Read` ↔ `Read` |
| MPSC     | `Send` ↔ `Read` + `Send` ↔ `Send` |
| MPMC     | `Read` и `Send` со всех сторон |

Где `SPSC` — Single Producer Single Consumer, `SPMC` — Single Producer Multiple Consumers, `MPSC` — Multiple Producers Single Consumer, `MPMC` — Multiple Producers Multiple Consumers.

**Синхронизация**

Во всех сценариях единообразно: один `mutex` вокруг контейнера + две `condition_variable` (`not_full`, `not_empty`); ожидание с проверкой предиката (защита от ложных пробуждений и потерянных уведомлений); `Close` будит всех ожидающих при остановке. *Оптимизация: two-lock (раздельные замки head/tail на связном списке) повышает параллелизм продюсер/консюмер — отложено.*

##### Интерфейс

**Параметры конкретного инстанса очереди**
| параметр    | описание |
| ----------- | -------- |
| MessageType | Тип сообщения |
| capacity    | Максимум сообщений в очереди |
| container   | Бэкенд хранения: `std::deque` (по умолчанию) или `std::list` |

**Публичные методы**
|  метод  | поведение         |
| ------- | ----------------- |
| Send    | Продюсер отправляет сообщение. Если очередь заполнена — блокируется поток продюсера, пока не освободится место |
| TrySend | Продюсер отправляет сообщение, если очередь не заполнена, иначе прокидывается ивент `OVERFLOW` |
| Read    | Консьюмер читает сообщение с головы (с удалением). Если очередь пуста — блокируется поток, пока не появится сообщение |
| TryRead | Консьюмер читает сообщение с головы (с удалением). Если очередь пуста — прокидывается ивент `EMPTY` |
| Size    | Текущее число сообщений в очереди |
| Stats   | Статистика очереди: число записей/чтений, потери (`0` для этой стратегии), время и количество блокировок, пиковый размер |
| Close   | Закрыть очередь: разбудить всех ожидающих; после закрытия `Read` отдаёт остаток и завершает, `Send` отклоняется |

| Метод                                   | Описание                                                                                                           |
| --------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| `Queue(size_t capacity)`                | Создаёт очередь с фиксированной ёмкостью. `capacity > 0`.                                                          |
| `void put(item)`                        | Добавляет элемент. Если очередь заполнена, вытесняет самый старый элемент. Поток не блокируется.                   |
| `T take(stop_token)`                    | Извлечение элемента с блокировкой.                                                                                 |
| `optional<T> take(timeout, stop_token)` | Извлечение элемента с блокировкой и таймаутом. При успехе (по `timeout`) возвращает элемент, иначе `std::nullopt`. |
| `std::optional<T> tryTake()`            | Извлечение элемента без блокировки. Если очередь пуста, возвращает `std::nullopt`.                                 |
| `std::optional<T> peek()`               | Возвращает копию головного элемента без извлечения. `std::nullopt`, если очередь пуста.                            |
| `size_t size() const`                   | Текущее количество элементов в очереди.                                                                            |
| `size_t capacity() const`               | Максимальная ёмкость очереди.                                                                                      |
| `bool empty() const`                    | `true`, если очередь пуста.                                                                                        |
| `bool full() const`                     | `true`, если очередь полностью заполнена.                                                                          |

| Исключение              | Описание                                        |
| ----------------------- | ----------------------------------------------- |
| `QueueCancelled`        | Бросается `take` при запросе отмены.            |
| `std::invalid_argument` | Бросается в конструкторе, если `capacity == 0`. |
